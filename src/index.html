Glossary

A

B

Box Model

a model that shows the properties of how space around an element is assigned and manipulated with CSS.

Breakpoint

The point at which a media query's condition becomes true.

C

CDN

Stands for Content Delivery Network.

A network of servers that delivers content to users that can be invoked by linking to its CDN location, such as Bootstrap CSS sheets.

CSS

Stands for Cascading Style Sheets.

D

Development Server

A development server is a type of server that is designed to facilitate the development and testing of programs, websites, software or applications for software programmers.

DOM

Stands for Document Object Model.

the DOM is an abstraction of structured text. It's a specific instance of any given HTML code that can be altered dynamically without changing the underlying code.

Elements in the HTML are considered 'nodes' of the DOM. The problem with the standard DOM is that for websites that load large amounts of objects (e.g. twitter, pinterest), the DOM then has too many nodes, which makes it difficult to interact with efficiently.

E

Environment

A combination of project dependencies within which you build your program.

F

Feature Branch

A branch specifically for testing a feature

Framework

The code that dictates the architectureof your project. Comes as a prebuilt tool.

Front-End

The portion of a website or program that is interacted with by the user.

Front-End Framework

Pre-built code that assists in creating the front end of a website or program.

Functional Programming

The use of basic functions as the building blocks of a program instead of objects

e.g. var saySomething = function(whatToSay){
alert(whatToSay);
}

This organizes the program around its actions.

Hot Module Replacement

Abbreviated as HMR.

When enabled, updated JavaScript modules are automatically bundled and served by webpack and its dev server.

G

H

I

Inheritance

the process by which child elements receive the same properties of the parent elements.

Inline Style

Style that is added directly into the HTML

e.g. '<\em>', <\strong>

These will always take precedence over CSS styles.

J

JSX

Stands for JavaScript XML

This is a preprocessor that adds special syntax and capabilities to JavaScript.

This is not inherently understood by the browser and must be transpiled before use.

K

L

Libary

Reusable pieces of code that are used without any hooks for you to extend it. The library will usually function on a single piece of functionality.

Loaders

The part of a program that is responsible for loading programs and libraries.

M

Media Query

A block of CSS rules that only apply when the screen the site/app is being viewed from is a certain size.

Media Types

The type of media device that the user is viewing content with.

Minfy

Minified code is the process of removing unnecessary characters (e.g. whitespace, comments, etc.) that humans need to read code but computers do not to reduce file size.

Module Bundler

A tool that collects source code and dependency code and concatenates it into a central file to decrease load time.

Most bundlers will also minify the code as well.

N

O

Object-Oriented Programming

The act of using objects to encapsulate data.

e.g. function Contact(first,last){
this.firstName = first;
this.lastName = last;
this.addresses = [];
}

Organizes the logic around the information managed by the application.

P

Package Manager

A tool to manage dependencies.

npm is the most common but yarn is also commonly used in the industry.

Plugins

a software component which adds a specific feature to an existing program.

Preprocessor

a program that modifies data to conform to the input requirements of another program.

e.g. Turning SASS into vanilla CSS

Property

The characteristics a CSS rule alters (e.g. color, font size)

Pure Functions

a function whose return values are determined using only input values and never alter external state or application data.

React components must always be pure and cannot alter their props; Props are read-only.

Q

R

Responsive Design

An approach to web design providing the best view and navigation for a given screen size or viewport.

Rule

A block of CSS that details the stylistic instructions applied to an HTML element.

Selector

The part of a CSS rule that determines which HTML element the rule applies to

e.g. class/id/tag

S

Self-Closing Element

an HTML element that doesn't require a closing tag.

T

Transpiler

a compiler that turns the source code in a given language into the equivalent source code in another language.

U

V

Value

The attribute a CSS rule applies to the specifed property.

Viewport

The tool a user is viewing content with. e.g. browser, tablet, phone, etc.

Virtual DOM

An abstraction of the HTML DOM. considered React's local and simplified copy of the HTML DOM.

It is faster and better than the regular DOM because it doesn't need to communicate with the browser in order to make its changes. React can calculate changes between the true DOM and the virtual DOM and update the true DOM to match the virtual.

W

Webpack

A type of module bundler.

Has flexible configuration and supports plugins and loaders.

Has a development server feature.
